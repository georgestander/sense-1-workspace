# Security Policy

## Reporting a vulnerability

Do not open public GitHub issues for security vulnerabilities.

Report security issues privately to the maintainer first. Include:

- a clear description of the issue
- affected area
- reproduction steps
- impact
- any suggested mitigation

Please avoid public disclosure until the issue has been reviewed and a fix or mitigation path is in place.

## Scope

Security reports are especially useful for:

- desktop file access and workspace boundaries
- approval and execution behavior
- plugin, skill, app, and automation surfaces
- local data handling
- update and packaging flows
